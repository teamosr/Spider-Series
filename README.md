## 实战1：小说爬取下载

这是我学习爬虫系列的第一个案例，主要实现了对特定小说网站分章节下载的功能。

### 功能特点
- 支持按“部/卷”精准定位下载。
- 自动过滤无效的 JavaScript 链接。
- 使用 `tqdm` 实现可视化下载进度条。
- 自动保存为 UTF-8 编码的 TXT 文件。

### 使用说明
1. 安装依赖：`pip install -r requirements.txt`
2. 运行脚本：`python main.py`

## 实战2：漫画爬取下载

### 使用网站
https://mycomic.com/comics/17965

### 项目总结
1. 核心思维：CSR (客户端渲染) 的识别与攻克

  - 现象：在浏览器里能看到内容，但在 requests 抓取的源码中，关键标签为空，或者只看到 <template x-for="..."> 这种占位符。
  - 本质：数据不是直接写在 HTML 里的，而是由 JavaScript 在浏览器端动态生成的。
  - 对策：引入 Selenium 自动化引擎。它能模拟真实浏览器行为，等待 JavaScript 执行完毕并“装修”好页面后，再通过
    driver.page_source 获取完整的 DOM 树。

2. 工具进阶：Selenium 与 WebDriver 的协同

  - 环境配置：学会了如何根据 Chrome 版本手动配置 chromedriver.exe，并理解了 Service 对象在 Python
    代码与浏览器驱动之间的“翻译官”角色。
  - 等待机制：理解了动态网页需要“渲染时间”，通过 time.sleep() 给了脚本与浏览器同步的机会。

3. 突破防御：破解“Referer”防盗链

  - 反爬手段：图片服务器会检查请求来源。如果直接访问图片 URL（无 Referer），服务器会判定为非法盗链并拒绝访问（403 错误）。
  - 破解方法：在 requests 请求头中手动构造 Referer 字段，伪装成“我是从漫画详情页点进来的”，从而成功获取二进制图片流。

4. 精准提取：应对“懒加载 (Lazy Loading)”

  - 细节观察：通过对比 F12 和 view-source，发现网站为了节省流量，只有前几张图在 src 属性，后面的图真实地址在 data-src 中。
  - 逻辑优化：学会了使用 img.get('data-src') or img.get('src') 这种具备回退机制的代码，保证了链接抓取的完整性。

5. 数据治理：二进制保存与物理排序

  - 文件操作：学会了使用 'wb' 模式保存二进制图片数据。
  - 逻辑连贯：为了保证漫画阅读不乱序，使用了数字格式化命名（如 001.jpg,
    002.jpg）。利用三位补零（:03d）解决了计算机文件排序中“10排在2前面”的经典问题。

6. 混合战术：Selenium + Requests 的配合

  - 分工合作：
      - Selenium：负责处理复杂的 JS 渲染、绕过环境检测、抓取动态生成的 URL 列表。
      - Requests：负责具体的、大批量文件的下载。
  - 意义：这种“混合战术”兼顾了 Selenium 的“破壁能力”和 Requests 的“下载速度”，是目前工业级爬虫的主流方案。

### 个人感悟

  - 眼见不一定为实：以后遇到抓不到的内容，先按 Ctrl + U 对比源码。如果源码没有，就立刻考虑 Selenium 或寻找 API。
  - 特征观察法：通过 if 'chapters' in url 这种关键词过滤，是处理杂乱 HTML 标签最高效的手段。
  - 环境一致性：这次遇到的 Chrome 148 版本问题提醒，保持驱动与浏览器版本大版本一致是 Selenium 运行的前提。

## 实战3：视频爬取爬取下载

### 爬取网站
https://1080zyk6.com/

### 项目流程
1. 首先是获取详细页链接，得到该视频的详细链接
2. 然后手动打开该详细的链接，提取出其中的m3u8链接，去除其中的一个来进行测试，获取该链接的响应内容，通过该响应内容的最后一行的输出可以获得具体的ts文件地址
3. 去除链接中不需要的一些元素，然后将响应内容的最后一行的地址加入获得ts文件的真正地址
4. 访问ts列表的地址，将这些ts下载到本地的文件夹中，之后再通过FFmpeg来将这些ts文件进行整合，最后获得完整的.mp4文件
### 个人收获
- 抓搜索的请求包方法：第一步输入要搜索的内容，第二步打开并清理Network，第三步点击搜索按钮；Network里第一个弹出的就是搜索的请求包
- 学会查看Network下的Paylaod（载荷，也就是发给服务器的数据）
    - 如果是GET的：会看到Query String Parameters，因为GET请求不被允许在“身体”里带数据，只能显式地展现在网页url中
    - 如果是POST的：会看到Form Data，因为POST会把发送的数据藏在请求体的内部，此处的数据就是隐藏的数据
    - 特殊情况下：会看到Query String Parameters和Form Data都存在，意味着请求是一个“带着挂件的POST”
- 视频网站在首次访问时会遇到验证码的输入，要想到使用Selenium+手动协作，在浏览器显示验证码的时候手动输入验证码来实现“系统安全验证”的解决
- 环境模拟：使用undetected-chromedriver抹除自动化特征，防止被Cloudflare等防火墙拦截，因为使用普通的Selenium(form selenium import webdriver)，浏览器会自动携带一个特殊的标记:window.navigator.webdriver=true，想Cloudflare这种高级防火墙，只要扫描到这个属性是true，会判定为机器人，而undetected-chromedriver会使得启动浏览器的时候，自动修改浏览器的底层代码，把那个属性改为underfined
- m3u8格式的视频：这种视频是由一个个ts视频片段组成，所以在获取m3u8的链接之后，还需要继续解析这个链接来获得最终.ts的列表
- 视频内容很大，所以不再使用单线程，使用ThreadPoolExecutor（线程池）
- 简单的把ts拼接成mp4文件可能无法正常播放，所以使用FFmpeg，它会自动修复视频的时间戳和元数据，解决“下载完打不开”或“快进时音画不同步”的问题
